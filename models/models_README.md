# MHARS Models

Model weight files are **not stored in git** because they are binary files that can be hundreds of megabytes. This folder should only contain the files listed below after you run the training scripts.

## How to regenerate all models

Run these commands from the project root with the venv activated:

```bash
# Stage 2 — ML models (Isolation Forest, LSTM, Autoencoder)
# Runtime: ~5 minutes on MacBook
cd stage2_ml
python run_stage2.py

# Stage 3 — PPO agent
# Runtime: ~30–40 minutes on MacBook CPU
cd ../stage3_ai
python run_stage3.py

# Stage 5 — Machine Adapter (requires Stage 2 and 3 complete)
# Runtime: ~5 minutes
cd ../stage5_adapter
python run_stage5.py
```

## How to download Phi-3 Mini (optional)

The LLM is optional — the system works without it using template-based alerts.

```bash
pip install llama-cpp-python

# Download the model file (~2.2 GB):
# https://huggingface.co/microsoft/Phi-3-mini-4k-instruct-gguf
# Direct link:
# https://huggingface.co/microsoft/Phi-3-mini-4k-instruct-gguf/resolve/main/Phi-3-mini-4k-instruct-q4.gguf

# Save to: models/Phi-3-mini-4k-instruct-q4.gguf
```

Then initialise MHARS with the path:

```python
from mhars import MHARS
system = MHARS(
    machine_type_id=0,
    llm_path="models/Phi-3-mini-4k-instruct-q4.gguf"
)
```

## Model architecture details

### ThermalLSTM
- Architecture: `LSTM(input=1, hidden=128) → Dropout(0.2) → Linear(128→1)`
- Input: sliding window of 12 normalized temperature readings
- Output: predicted next temperature (normalized)
- Reference: Kalla & Smith (2024), *Information* 16(9), 737

### ThermalAutoencoder
- Architecture: `Linear(12→6→3)` encoder, `Linear(3→6→12)` decoder
- Trained on normal-only data
- Anomaly score = reconstruction error / threshold (95th percentile)
- Reference: Gutiérrez et al. (2024), *Aerospace* 11(7), 523

### PPO Agent
- Library: Stable-Baselines3 v2.8
- Algorithm: Proximal Policy Optimization (`clip_range=0.2`)
- Training: 500,000 timesteps across 4 parallel environments
- Reference: Schulman et al. (2017), arXiv:1707.06347
